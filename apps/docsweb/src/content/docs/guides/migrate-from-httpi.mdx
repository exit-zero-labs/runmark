If you already use `httpi`, the move to **Runmark** is a clean rename. Update
the package, command, repo references, and on-disk project folder. The workflow
model stays the same.

## What changed

| Surface | Old | New |
| --- | --- | --- |
| product name | `httpi` | `Runmark` |
| npm package | `@exit-zero-labs/httpi` | `@exit-zero-labs/runmark` |
| CLI command | `httpi` | `runmark` |
| repository | `exit-zero-labs/httpi` | `exit-zero-labs/runmark` |
| project folder | `httpi/` | `runmark/` |
| docs domain | `httpi.exitzerolabs.com` | `runmark.exitzerolabs.com` |

## What stays the same

- your request, run, env, and block file formats
- the workflow model
- CLI and MCP behavior
- session and artifact semantics
- schema-driven YAML authoring

## Recommended migration steps

1. Replace the global install:

```bash
npm uninstall -g @exit-zero-labs/httpi
npm install -g @exit-zero-labs/runmark
runmark --version
```

2. Update MCP client config:

```json
{
  "mcpServers": {
    "runmark": {
      "command": "runmark",
      "args": ["mcp"]
    }
  }
}
```

If you prefer `npx`, use `@exit-zero-labs/runmark` instead of
`@exit-zero-labs/httpi`.

3. Rename the project folder inside each repository:

```bash
mv httpi runmark
```

4. Update scripts, CI jobs, README snippets, and shell history assumptions from:

- `httpi` -> `runmark`
- `@exit-zero-labs/httpi` -> `@exit-zero-labs/runmark`
- `exit-zero-labs/httpi` -> `exit-zero-labs/runmark`
- `httpi/` -> `runmark/`

5. Re-run your normal workflow against the renamed project:

```bash
runmark validate
runmark describe --run smoke
runmark run --run smoke
```

## No compatibility mode

This rename ships as a clean break. There is no long-term dual naming.
Existing projects should move to `runmark/`, and old `httpi` package or command
references should be treated as historical only.

## FAQ

### Is Runmark a new product?

No. Runmark is the renamed `httpi` product.

### Is httpi being discontinued?

Yes as a name, no as a product. The workflow model, codebase, and interfaces
continue under the Runmark name.

### Do I need to change my request files?

Usually not. The file contents stay the same; the main change is that project
discovery now expects `runmark/config.yaml`, and the on-disk folder should be
`runmark/`.

### Do I need to update MCP configuration?

Yes. Update both the command and package name to `runmark` /
`@exit-zero-labs/runmark`.

### Why do some docs still say httpi?

Only migration docs and historical notes should still mention `httpi`, and
only when they are mapping old identifiers to new ones.
