Runmark is a file-based HTTP workflow runner for repositories. It keeps tracked
request intent in `runmark/`, Git-ignored runtime state in
`runmark/artifacts/`, and exposes the same execution model through both a CLI
and an MCP server.

## Install

One package, two surfaces:

```bash
npm install -g @exit-zero-labs/runmark
```

- `runmark` — the CLI for humans and scripts
- `runmark mcp` — the stdio MCP server for coding agents

## Start a project

```bash
mkdir demo-api && cd demo-api
runmark init
```

That gives you a small but real starting point: one environment, one request,
one run, and schema-aware YAML files.

After `init`, the normal first loop is:

```bash
# edit runmark/env/dev.env.yaml so baseUrl points at your service
runmark validate
runmark describe --run smoke
runmark run --run smoke
```

If the flow needs local secrets, add them under `runmark/artifacts/secrets.yaml`:

```yaml
devPassword: swordfish
apiToken: sk_test_123
```

Tracked files can reference `{{secrets.alias}}` or `$ENV:NAME`, but secret
literals should stay out of `runmark/`.

## Project layout

```text
demo-api/
└── runmark/
    ├── config.yaml
    ├── env/
    │   └── dev.env.yaml
    ├── requests/
    │   └── ping.request.yaml
    ├── runs/
    │   └── smoke.run.yaml
    └── artifacts/
        ├── secrets.yaml
        ├── sessions/
        └── history/
```

| Path | Purpose |
| --- | --- |
| `runmark/` | Git-tracked requests, runs, envs, blocks, body templates, and the Git-ignored runtime subtree |
| `runmark/artifacts/` | Local secrets, session state, locks, and captured request artifacts |

## Core commands

| Goal | Command |
| --- | --- |
| scaffold a project | `runmark init` |
| discover requests, runs, envs, or sessions | `runmark list` |
| validate before execution | `runmark validate` |
| inspect a request or run shape | `runmark describe` |
| inspect resolved values and provenance | `runmark explain variables` |
| execute a request or run | `runmark run` |
| inspect a paused or failed session | `runmark session show` |
| inspect captured evidence | `runmark artifacts list` / `runmark artifacts read` |
| continue a paused or failed workflow | `runmark resume` |
